‚óè Voice Command Hub for Odoo - Module Overview

  This is a production-grade Odoo addon that enables users to execute
  business operations through natural language voice/text commands.

  Core Purpose

  Transform spoken or typed commands like "Toufik buy 5 chocolates from me"
  into actual Odoo operations (creating sales orders, managing inventory,
  registering payments, etc.) with full safety validation and audit trails.

  How It Works - Key Flow

  1. User Input ‚Üí Text or voice command received
  2. Intent Recognition ‚Üí AI matches command to business action (sale,
  purchase, inventory, etc.)
  3. Entity Extraction ‚Üí Identifies customer names, products, quantities,
  dates, amounts
  4. Interactive Slot Filling ‚Üí Asks follow-up questions for missing
  information
  5. Risk Assessment ‚Üí Classifies as low/medium/high risk
  6. Dry-run Simulation ‚Üí Shows preview of what will happen (NO real
  changes)
  7. User Confirmation ‚Üí Requires approval for medium/high risk actions
  8. Real Execution ‚Üí Creates/updates actual Odoo records
  9. Audit Trail ‚Üí Complete logs with timestamps, before/after states

  Main Components

  1. NLU Engine (services/voice_intent_router.py)

  - Multi-strategy intent matching (exact ‚Üí fuzzy)
  - Keyword boosting for domain specificity
  - Confidence scoring (0.0-1.0)

  2. Entity Extractor (services/voice_slot_filler.py)

  - Extracts: partners, products, quantities, money, dates, booleans
  - Fuzzy matching for partner/product names
  - Pattern-based extraction (regex for emails, phones, amounts)

  3. Intent Handlers (Pluggable architecture)

  - SaleCreateHandler: Create & confirm sales orders, optionally invoice
  - PurchaseCreateHandler: Create purchase orders
  - InventoryAdjustHandler: Update stock quantities
  - CrmLeadHandler: Create leads/opportunities
  - InvoicePaymentHandler: Register invoice payments

  4. Session Management (models/voice_command_session.py)

  State machine: collecting ‚Üí ready ‚Üí executed/aborted

  5. HTTP API (controllers/main.py)

  - POST /voice/command - Submit new command
  - POST /voice/command/<id>/execute - Execute confirmed command
  - POST /voice/command/<id>/fill_slot - Provide missing information

  Key Features

  ‚úÖ Safety First: Dry-run simulations, confirmation for risky actions,
  database savepoints
  ‚úÖ Full Audit: Every action logged with complete history
  ‚úÖ Interactive: Asks questions for missing info
  ‚úÖ Secure: Respects Odoo ACL, no privilege escalation
  ‚úÖ Extensible: Easy to add custom intent handlers
  ‚úÖ Multi-domain: Sales, Purchase, Inventory, Accounting, CRM
  ‚úÖ Learning: Feedback system improves accuracy over time
  ‚úÖ Configurable: Fuzzy matching threshold, confirmation policies, defaults


  Example Workflow

  Command: "Create sale order for Azure Interior with 10 desks"

  1. ‚úì Intent detected: sale_create (confidence: 0.92)
  2. ‚úì Extracted slots:
    - partner: "Azure Interior" (ID: 14)
    - product_lines: [{"product": "Desk", "qty": 10}]
  3. ‚úì Risk level: Medium (financial impact)
  4. ‚úì Simulation shows: SO total = $2,400.00
  5. ‚úì User confirms
  6. ‚úì Creates sale.order SO/2024/0025
  7. ‚úì Audit log: All steps timestamped

  Technical Stack

  - Odoo ORM: All data operations
  - Built-in NLU: No external dependencies (optional spaCy/Transformer
  integration)
  - Python stdlib: re, difflib, json, datetime
  - Savepoint-based transactions: Auto-rollback on errors

  Security Model

  - User authentication required (auth='user')
  - Odoo ACL respected throughout
  - Group-based intent restrictions
  - No sudo() except for reading system config
  - Complete audit trail for compliance

  This module is production-ready with comprehensive error handling, full
  audit capabilities, and a flexible architecture for extending to custom
  business operations.
